# Contributing to PS2-EtherDrive

## Introduction

Thank you for your interest in the PS2-EtherDrive project.

This repository serves as a public development, documentation, research, and support hub for the project.

While some portions of the project may remain closed during development and early production phases, community feedback, testing, research, and technical discussion are greatly appreciated.

---

# Ways to Contribute

Community contributions may include:

- Technical research
- Documentation improvements
- Testing feedback
- Compatibility reports
- Installation observations
- Thermal testing
- Network performance testing
- Bug reporting
- Feature suggestions

---

# Reporting Issues

GitHub Issues should be used for:

- Bug reports
- Documentation corrections
- Compatibility observations
- Installation problems
- Firmware issues
- Hardware concerns

When possible, include:

- Console revision
- Hardware revision
- Firmware version
- Storage device used
- Detailed reproduction steps
- Photos or screenshots if applicable

---

# Discussions and Feedback

GitHub Discussions are encouraged for:

- General project discussion
- Ideas and suggestions
- Research sharing
- Development conversation
- Community testing notes

Constructive discussion is always appreciated.

---

# Documentation Contributions

Documentation improvements are welcome.

Helpful contributions may include:

- Correcting technical inaccuracies
- Improving readability
- Expanding installation notes
- Improving troubleshooting guidance
- Organizing research material

---

# Hardware and Firmware Contributions

At this time:

- Production hardware files
- Gerber files
- Manufacturing files
- Firmware source code

may remain private during development and early production phases.

Community feedback and external research are still extremely valuable and appreciated.

---

# Contribution Philosophy

The goal of the project is to build a well-documented and community-supported hardware platform while balancing:

- Long-term sustainability
- Development cost recovery
- Product quality
- Community collaboration

---

# Testing Contributions

Testing contributions are especially valuable.

Areas where community testing may eventually help include:

- Console compatibility
- Storage compatibility
- WiFi behavior
- SMB performance
- UDPBD testing
- Thermal validation
- Long-term stability

---

# Respectful Collaboration

Please keep discussion:

- Respectful
- Constructive
- Technical
- Helpful

The goal is building a positive and useful development environment for everyone involved.
